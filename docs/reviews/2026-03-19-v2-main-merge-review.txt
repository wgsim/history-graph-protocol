1. High: inactive COMPLETED operations are skipped by reconciler missing-blob checks.
   Code:

2. High: releasing one lease demotes the root to long_term even when another active lease still exists.
   Code:
   - /home/woogwangsim/git_repos/history-graph-protocol/.worktrees/feat-v2-memory-tier/src/hgp/lease.py#L29
   - /home/woogwangsim/git_repos/history-graph-protocol/.worktrees/feat-v2-memory-tier/src/hgp/server.py#L245
   Detail:
   - LeaseManager only auto-releases prior ACTIVE leases for the same agent + subgraph.
   - Different agents can still hold ACTIVE leases on the same root.
   - hgp_release_lease() does not check for other ACTIVE leases before demoting the root tier.
   Reproduction observed:
   - one lease remained ACTIVE
   - root memory_tier became long_term

3. Medium: acquire -> create_operation(lease_id=...) -> auto-release leaves the root stuck in short_term.
   Code:
   - /home/woogwangsim/git_repos/history-graph-protocol/.worktrees/feat-v2-memory-tier/src/hgp/server.py#L128
